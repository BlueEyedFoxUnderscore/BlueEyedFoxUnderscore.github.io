Robotics

  Robotics is both an engineering challenge and a coding challenge. The creation of a machine to accomplish a certain goal is almost as hard as telling that machine how to accomplish said goal. While building and creating a machine are not usually things that I prefer to do, I have participated in the creation of the software side of some robots. I have several repos detailing the process of creating and revising that code.

  While it may seem overly complicated or boring, the process of creating a logic tree or code for a robot is actually quite interesting.

Games

  Games are actually very complex to code, as you have to manually input everything from object placement to entity pathing. One of my specialites slash areas of expertise is procedural, or dimensional, animation, which allows for the simplification of the animation process by the application of procedural generation, which means that you pop some numbers into an equation and get some numbers out, and then use those numbers as angles and lengths to make an animation.
